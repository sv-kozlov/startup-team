# Discovery Question Bank

Use this file only when the first user request is vague, the readiness score is below 80, or a specific score area is weak.

Ask 2-4 questions per turn. Use the user's conversation language.

## Start Questions

1. What problem are we solving, and for whom?
2. What outcome would make this successful?
3. What is explicitly out of scope?

## Business Value

- Why is this needed now?
- What user or business pain exists today?
- What happens if this is not built?
- Which metric should improve?

## Users and UX

- Who is the primary user?
- What is the main user workflow?
- Where does this fit into the current user journey?
- Are there accessibility, localization, or platform constraints?

## Scope

- What is the MVP?
- What must be excluded from this release?
- What can move to a later phase?
- What are the top 3 priorities?

## Metrics

- What is the baseline today?
- What is the target after launch?
- How will we measure it?
- Who owns the metric?

## Technical Constraints

- What systems must integrate?
- Are there performance, security, privacy, or compliance constraints?
- Are there data migration or compatibility requirements?

## Risks and Assumptions

- What assumptions are we making?
- What could make this fail?
- Which assumptions need validation before delivery?

## AI Features

- What inputs will the AI receive?
- What outputs are acceptable or unacceptable?
- How will quality be evaluated?
- What fallback is required when confidence is low?
- What data must not be sent to the model?

## Stop Rules

Stop discovery and draft when:

- readiness score is 80+;
- problem, users, MVP, non-goals, metrics, and top risks are known;
- the user asks for a draft now.

If still below 80 but the user wants progress, draft with `TBD` and `Open Questions`.
