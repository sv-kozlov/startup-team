# Sources — Tech Lead

Consolidated external sources referenced across this subagent's skills. Organized by priority. Priority 1 sources are canonical and override Priority 2–3 when guidance conflicts.

## Priority 1 — Canonical books and engineering practices

Authoritative for technical leadership, engineering management, and team-level practice. Cited across method-family and lead-family skills.

- Camille Fournier: The Manager's Path — O'Reilly, 2017. Core reference for tech lead progression, mentoring, and team management.
- Will Larson: An Elegant Puzzle: Systems of Engineering Management — Stripe Press, 2019. Team sizing, capacity, technical debt strategy, cross-team alignment.
- Will Larson: Staff Engineer: Leadership beyond the Management Track — Larson, 2021. Principal and staff-level technical leadership, architectural ADRs, cross-org influence.
- Patrick Lencioni: The Five Dysfunctions of a Team — Jossey-Bass, 2002. Team dynamics baseline for leadership practice.
- Google Engineering Practices: Code Review — https://google.github.io/eng-practices/review/
- Google SRE Book: Postmortem Culture — https://sre.google/sre-book/postmortem-culture/
- ADR GitHub organization — https://adr.github.io/
- Michael Nygard: Documenting Architecture Decisions — https://www.cognitect.com/blog/2011/11/15/documenting-architecture-decisions

## Priority 2 — Authoritative external orientation

Engineering quality, hiring, observability, and reliability practice. Cited in specific skills.

- Lara Hogan: Resilient Management — A Book Apart, 2019. Feedback delivery, growth conversations, 1:1 structure.
- Charity Majors: On-call and observability practice — https://charity.wtf/
- Tanya Reilly: The Staff Engineer's Path — O'Reilly, 2022. Cross-team alignment, project-level influence, technical strategy.
- Google SRE Workbook — https://sre.google/workbook/table-of-contents/
- DORA State of DevOps Reports — https://dora.dev/research/
- Martin Fowler: Technical Debt — https://martinfowler.com/bliki/TechnicalDebt.html
- Ward Cunningham: Technical Debt original framing — https://wiki.c2.com/?WardExplainsDebtMetaphor
- The Twelve-Factor App — https://12factor.net/
- Jeff Atwood: Code Review Best Practices — https://blog.codinghorror.com/code-reviews-just-do-it/
- Ron Lichty, Mickey Mantle: Managing the Unmanageable — Addison-Wesley, 2012. Engineering team hiring and interview design.
- Gergely Orosz: The Pragmatic Engineer Newsletter — https://newsletter.pragmaticengineer.com/
